# GATE A — The second clock (`test19_second_clock.py`)

## A.1 The vulnerability

`clock_fit.py` fits a **linear** clock:

```python
LinearClock:  age = w·x + b
```

But the **published Fleischer clock is a nonlinear LDA ensemble.** You approximated it linearly.

A reviewer's first question will be:

> *"You concluded deep models cannot beat linear baselines on ΔAge. But your ΔAge target was
> generated by a linear model you fitted yourself. That is close to circular."*

**It is not fully circular** — the per-donor control-inference problem is genuinely nonlinear, and
that is what defeated the flexible models in Test 6. **But Test 6's result is conditional on a
linear clock, and that conditionality is currently unstated anywhere in the writeup.**

## A.2 The test

`AgingClock` is an ABC with `predict_age(expr, genes)`. Implement a **nonlinear** subclass:

```python
# src/cellfate/data/aging.py
class LDAEnsembleClock(AgingClock):
    """Nonlinear transcriptomic clock, closer to the published Fleischer construction:
    an ensemble of linear discriminants over age bins, rather than a single linear fit."""

    def __init__(self, bin_edges, discriminants, bin_centers):
        self._bins = bin_edges
        self._lda = discriminants        # one LDA per age bin
        self._centers = bin_centers

    def predict_age(self, expr: np.ndarray, genes: list[str]) -> np.ndarray:
        # posterior over age bins, then expectation -> a genuinely NONLINEAR age readout
        post = np.stack([d.predict_proba(expr)[:, 1] for d in self._lda], axis=1)
        post = post / post.sum(axis=1, keepdims=True)
        return post @ self._centers
```

**Then:**
1. regenerate ΔAge targets with `LDAEnsembleClock`
2. **rerun Test 6 unchanged** (`test6_beat_ridge.py`) — ridge vs boosted trees vs random forest
   vs kernel RBF

## A.3 Decision table

| Outcome | Consequence for the paper |
|---|---|
| **Ridge still wins** | **The claim gets much stronger** — it now holds for a nonlinear target too, and the circularity objection is answered with data |
| **A flexible model wins** | The finding becomes *"deep ties linear **when the clock is linear**, and wins when it is not"* — **more novel, more interesting**, and a better paper |

> **Either result improves the paper. The only bad outcome is not running it.**

## A.4 What must not change

Rerun Test 6 **exactly as written** — same models, same hyperparameters, same folds. The only
variable is the clock. Changing anything else makes the comparison uninterpretable.

| fold | model PR-AUC | logreg PR-AUC |

## 4. Suggested framing

> **"Donor-level calibration is the binding constraint for decision-making in reprogramming
> rejuvenation."**

MaxToki's existence **helps** this framing: a foundation model trained on 175M cells still has to
be calibrated for one donor before anyone makes a withdrawal decision. Your work quantifies how
much that matters and what it costs to fix. **A companion finding, not a competitor.**

## 5. Limitations that must appear in the manuscript

1. **n = 6 donors**, two of which (N2, N3) are outliers driving the aggregates
2. **One dataset**, no independent replication
3. **The core cross-perturbation thesis is untested** — the data cannot pose it
4. **The fate edge rests on one fold** (Y1) — even after Gate B, n=5 with 4 saturated
5. **No wet-lab validation** of any prediction
6. **Stage 2 requires reference cells** — a protocol change, not a free fix
7. **Population intervals assume independent cells** — a lower bound on true uncertainty

## 6. Realistic venue

A solid **Bioinformatics / Genome Biology / PLOS Computational Biology** paper, or a strong
preprint. **Not** *Nature Methods* without more donors and a replication dataset — and being
honest about that now saves a painful review cycle.
